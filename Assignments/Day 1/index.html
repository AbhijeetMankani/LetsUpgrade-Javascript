var and let are used for mutable variable declaration, while, const declares an immutable variable or a constant
assigning a value to a constant(const-declared variable) will cause an error to occur
var was there in JS since ES5 while let and const arrived in ES6

scope of var:
var can be global or function scoped

The problem with var:
var-variables can be re-declared, and thius we cannot have the same variable name inside and outside of a function for 2 different variable, they will be considered as the same variable. (Code1:)

let:
let is now preferred for variable declaration. It's no surprise as it comes as an improvement to var declarations. It also solves the problem with var that we just covered.

scope of let:
let is block scoped. I.E. it can only be used inside the block in which it was declared(block is anything inside curly braces{} ) (Code2:)

let can be updated but not re-declared.
Just like var, a variable declared with let can be updated within its scope. Unlike var, a let variable cannot be re-declared within its scope.

However, if the same variable is defined in different scopes, there will be no error.
